# Email Deliverability in the Age of AI: Best Practices for Autonomous Outreach

AI agents can send 2,000 personalized emails per month. But only if they reach the inbox. Here's how to maintain 95%+ deliverability at scale in 2026.

## The Deliverability Problem

**2024 Problem:** Send 1,000 generic emails → 60% reach inbox
**2026 Solution:** Send 1,000 hyper-personalized emails → 95% reach inbox

The difference: Quality, infrastructure, and AI-native best practices.

## The Deliverability Stack

```typescript
import { configureDeliverability } from '@sales-builder/email'

export const deliverabilityConfig = configureDeliverability({

  infrastructure: {

    domains: {

        policy: 'p=quarantine', // Start with quarantine, move to reject

        primary: 'mx1.resend.com',
        backup: 'mx2.resend.com'
      }

    // Email service provider
    esp: {
      provider: 'resend', // Or sendgrid, postmark

  // Warmup strategy
  warmup: {
    enabled: true,

      day1: { emails: 10, toWhom: 'known-good-addresses' },
      day2: { emails: 20, toWhom: 'known-good-addresses' },
      day3: { emails: 30, toWhom: 'known-good-addresses' },
      day7: { emails: 50, toWhom: 'mix-of-known-and-cold' },
      day14: { emails: 100, toWhom: 'normal-targeting' },

      day30: { emails: 500, toWhom: 'normal-targeting' },
      day45: { emails: 1000, toWhom: 'normal-targeting' },

    monitoring: {
      // Watch for issues during warmup
      track: ['bounce-rate', 'spam-complaints', 'open-rate'],
      alerts: [
        { metric: 'bounce-rate', above: 0.05, action: 'pause-warmup' },
        { metric: 'spam-complaints', above: 0.002, action: 'pause-warmup' },
        { metric: 'open-rate', below: 0.15, action: 'review-content' }
      ]
    }
  },

  // List hygiene
  listHygiene: {
    // Verify emails before sending
    verification: {
      provider: 'zerobounce', // Or hunter.io, neverbounce
      verify: ['syntax', 'mx-record', 'smtp', 'catch-all'],
      onlyEmailVerified: true // Never send to unverified
    },

    // Remove bad addresses
    suppression: {
      automatic: [
        'role-based-emails', // info@, support@, admin@
        'disposable-emails', // mailinator, temp-mail, etc
        'previous-bounces',
        'previous-spam-complaints',
        'previous-unsubscribes'
      ],

      patterns: [
        /noreply@.*/,
        /do-not-reply@.*/,
        /notifications@.*/,
        /alerts@.*/
      ]
    },

    // Handle responses
    responses: {
      unsubscribe: 'immediate-removal',
      bounce: {
        hard: 'immediate-removal',
        soft: 'retry-3x-then-remove'
      },
      spamComplaint: 'immediate-removal-and-alert'
    }
  },

  // Content quality
  content: {
    // AI generates high-quality content
    personalization: {
      required: true,
      minPersonalizationScore: 0.7, // 70% unique per email

      elements: [
        'recipient-name',
        'company-name',
        'recent-activity-or-news',
        'specific-pain-point',
        'custom-value-prop'
      ]
    },

    // Avoid spam triggers
    spamPrevention: {
      forbidden: [
        'all-caps-subject-lines',
        'excessive-exclamation-marks',
        'misleading-subjects',
        'hidden-text',
        'massive-images',
        'URL-shorteners'
      ],

      limits: {
        exclamationMarks: { max: 1 },
        allCapsWords: { max: 2 },
        links: { max: 3 },
        images: { max: 1 },
        charactersPerLine: { max: 80 }
      },

      required: [
        'plain-text-version',
        'unsubscribe-link',
        'physical-address',
        'clear-sender-identity'
      ]
    },

    // Optimal formatting
    formatting: {
      html: {
        simpleOnly: true, // No complex layouts
        inlineCSS: true,
        noJavaScript: true,
        darkModeOptimized: true
      },

      plainText: {
        required: true, // Always include plain text version
        width: 72, // Characters per line
        encoding: 'UTF-8'
      },

      signature: {
        include: true,
        elements: ['name', 'title', 'company', 'website', 'unsubscribe']
      }
    }
  },

  // Sending behavior
  sendingBehavior: {
    // Don't send like a bot
    volume: {
      dailyLimit: 100, // Conservative start
      rampUp: 'gradual-over-60-days',
      maxDailyLimit: 500 // Even at scale, don't blast
    },

    timing: {
      // Send during business hours in recipient timezone
      hours: { start: 8, end: 18 },
      days: ['mon', 'tue', 'wed', 'thu', 'fri'],
      timezone: 'recipient-local', // Not sender's
      randomize: {
        enabled: true,
        variance: '±30min' // Don't send all at exactly 9 AM
      }
    },

    throttling: {
      // Spread sends throughout day
      emailsPerHour: 20,
      emailsPer5Min: 5,
      delayBetweenEmails: randomBetween(30, 90) // seconds
    },

    engagement: {
      // Prioritize engaged recipients
      boostPriority: [
        'previous-opens',
        'previous-replies',
        'previous-clicks'
      ],
      lowerPriority: [
        'never-opened-3x',
        'no-engagement-30days'
      ]
    }
  },

  // Monitoring & optimization
  monitoring: {
    realtime: {
      track: [
        'delivery-rate',
        'bounce-rate',
        'spam-complaint-rate',
        'open-rate',
        'reply-rate',
        'unsubscribe-rate'
      ],

      thresholds: {
        deliveryRate: { min: 0.95, alert: 'immediate' },
        bounceRate: { max: 0.05, alert: 'immediate' },
        spamRate: { max: 0.001, alert: 'urgent' },
        openRate: { min: 0.25, alert: 'review-content' }
      }
    },

    optimization: {
      abTest: {
        enabled: true,
        elements: ['subject-lines', 'send-times', 'personalization-approaches'],
        winner: 'auto-select-after-100-sends'
      },

      learning: {
        from: ['high-open-emails', 'high-reply-emails'],
        apply: ['subject-patterns', 'content-structures', 'send-timing']
      }
    }
  }
})
```

## Critical Deliverability Factors

### 1. Domain Reputation (30% of deliverability)

**What hurts:**
- Sending cold email from primary domain
- New domain, no history
- Sudden volume spikes
- High bounce rates
- Spam complaints

**What helps:**
```typescript
const domainStrategy = {
  primary: {
    domain: 'sales-builder.com',
    use: 'website-and-transactional-only',
    protect: 'never-use-for-cold-outbound'
  },

  outbound: {
    domains: [
      'mail.sales-builder.com',
      'outreach.sales-builder.com',
      'hello.sales-builder.com'
    ],
    warmup: '60-days-before-full-volume',
    monitoring: 'continuous-reputation-tracking'
  }
}
```

### 2. Email Content (25% of deliverability)

**Spam triggers to avoid:**
```typescript
const spamTriggers = {
  subject: [
    'FREE!!!',
    'Click here NOW',
    'LIMITED TIME OFFER',
    'Make money fast',
    'Dear friend' // Generic
  ],

  body: [
    'Remove from list', // Use proper unsubscribe
    'This is not spam', // Ironically, spammy
    'Click below', // Too many CTAs
    'XXX', 'Viagra', 'Casino', // Obvious spam words
    'Act now', // Urgency
    'As seen on', // False authority
  ],

  formatting: [
    'ALL CAPS PARAGRAPHS',
    'Hidden text', // Same color as background
    'Massive images', // Image-only emails
    'URL shorteners', // bit.ly, etc
    'Attachment on first email' // Red flag
  ]
}
```

**Good email structure:**
```
Subject: Quick question about [Company]'s sales automation

Hi [Name],

Saw [Company] recently [specific recent activity]. Congrats on [specific achievement]!

Quick question: How are you handling [specific challenge related to their role]?

[One sentence about our solution specific to their situation]

Worth a quick chat? [One clear CTA]

Best,
[Your Name]
[Title]
[Company]

---
Unsubscribe: [link]
[Physical address]
```

### 3. Infrastructure (20% of deliverability)

**DNS Configuration:**
```bash
# SPF (Sender Policy Framework)
TXT record: v=spf1 include:_spf.resend.com ~all

# DKIM (DomainKeys Identified Mail)
TXT record: resend._domainkey IN TXT "v=DKIM1; k=rsa; p=[public key]"

# DMARC (Domain-based Message Authentication)
TXT record: v=DMARC1; p=quarantine; rua=mailto:dmarc@sales-builder.com

# MX Records
MX priority 10: mx1.resend.com
MX priority 20: mx2.resend.com
```

### 4. Engagement (15% of deliverability)

**Gmail/Outlook prioritize engaged recipients:**

## Common Mistakes That Kill Deliverability

### Mistake 1: Sending Too Much Too Fast
```typescript
// Bad: Day 1 send 1000 emails
const badWarmup = {
  day1: 1000 // SPAM FOLDER
}

// Good: Gradual ramp over 60 days
const goodWarmup = {
  week1: 10/day,
  week2: 25/day,
  week4: 50/day,
  week8: 100/day,
  week12: 200/day
}
```

### Mistake 2: Not Verifying Emails
```
Unverified list:
- 15% bounces
- Reputation destroyed
- IP blacklisted
- Deliverability tanks

Verified list:
- <2% bounces
- Reputation protected
- Normal operation
- 95% deliverability
```

### Mistake 3: Using Primary Domain
```
sales-builder.com (primary):
- Used for website
- Used for transactional emails
- ALSO used for cold outbound ❌

Result:
- Cold email spam complaints hurt transactional delivery
- Welcome emails land in spam
- Password resets land in spam
- Business destroyed

Solution:
- outreach.sales-builder.com for cold email
- Protect primary domain
```
