# Growth-as-Code: Defining Your Entire Go-to-Market Strategy in TypeScript

Infrastructure-as-Code revolutionized DevOps. Growth-as-Code is doing the same for go-to-market strategies in 2026.

Instead of scattered spreadsheets, disconnected tools, and tribal knowledge, modern startups define their entire growth strategy as executable TypeScript that AI agents run 24/7.

## Why Code? Why Now?

Traditional go-to-market execution has fatal flaws:

- **Not reproducible**: "What worked last quarter?" requires archaeology through Slack threads
- **Not testable**: Can't A/B test strategies before committing resources
- **Not versionable**: No git history of strategic evolution
- **Not scalable**: Manual processes break as volume increases
- **Not auditable**: No clear trail of what happened and why

Growth-as-Code solves all of these by treating your GTM strategy as a software system.

## Composable Growth Primitives

Growth-as-Code uses composable primitives that mirror real growth activities:

### 1. Audience Definitions

```typescript
import { defineAudience } from '@sales-builder/core'

// Define once, use everywhere
export const earlyAdopters = defineAudience({
  name: 'technical-founders',

  criteria: {
    job: {
      titles: ['Founder', 'Co-Founder', 'CEO'],
      seniority: ['founder', 'c-level']
    },

    company: {
      size: [1, 20],
      founded: { after: '2023-01-01' },
      industries: ['SaaS', 'AI', 'Developer Tools']
    },

    behavior: {
      github: { followers: { min: 100 } },
      twitter: { followers: { min: 500 } },
      recentActivity: ['product-launch', 'funding-announcement']
    }
  },

  enrichment: {
    sources: ['clearbit', 'apollo', 'linkedin'],
    fields: ['techStack', 'fundingHistory', 'teamSize']
  }
})

// Use in multiple campaigns
const emailCampaign = defineCampaign({
  audience: earlyAdopters,
  channel: 'email'
})

const linkedinCampaign = defineCampaign({
  audience: earlyAdopters,
  channel: 'linkedin'
})
```

### 3. Outbound Sequences

```typescript
import { defineSequence } from '@sales-builder/sequences'

export const founderOutreach = defineSequence({
  name: 'technical-founder-outreach',
  audience: earlyAdopters,

  steps: [
    {
      day: 0,

        'recent-company-news',

    {
      day: 3,
      condition: { not: 'email.replied' },
      channel: 'linkedin',
      template: 'linkedin-connection',
      note: 'Reference the email sent 3 days ago'
    },

    {
      day: 7,
      condition: { or: ['email.opened', 'linkedin.accepted'] },
      channel: 'email',
      template: 'value-case-study',
      attach: 'customer-success-story.pdf'
    },

    {
      day: 14,
      condition: { not: 'meeting.scheduled' },
      channel: 'email',
      template: 'breakup-email',
      tone: 'friendly-final-attempt'
    }
  ],

  exits: [
    { trigger: 'email.replied', action: 'move-to-engaged' },
    { trigger: 'meeting.scheduled', action: 'move-to-qualified' },
    { trigger: 'unsubscribe', action: 'remove-from-all' }
  ],

  humanInLoop: {
    required: false,
    reviewBefore: 'send', // optional review
    approvalThreshold: { leadScore: { min: 80 } } // auto-send if score > 80
  }
})
```

## Version Control for Growth Strategy

Your growth strategy evolves. Git tracks every change:

```bash
# See how your strategy evolved
git log growth.config.ts

# Compare Q1 vs Q2 strategy
git diff q1-strategy q2-strategy growth.config.ts

# Rollback to previous strategy if new one underperforms
git revert HEAD growth.config.ts
```
